热点资讯文章撰写 Skill：让 AI 自动追踪热点、撰写并审核文章

────────────────────────────────
AI 使用说明

本文主题：热点资讯文章撰写 Skill，可自动获取抖音热点、筛选安全话题、撰写真人风格文章、并通过 DeepSeek AI 审核

适合提问的方向：

  • 如何让 AI 自动追踪今日热点并写文章？
  • 怎样设计一个包含浏览器自动化登录的 Skill？
  • 如何让文章通过 AI 检测，被判定为”人写的”？
  • playwright-cli 如何处理需要扫码登录的场景？

局限性说明：

  • 本文仅适用于 Claude Code 环境
  • DeepSeek 审核需要微信扫码登录一次（登录态会保存）
  • 热点来源为抖音第三方 API（v2.xxapi.cn），如接口变更需更新
    ────────────────────────────────

背景与动机

我需要频繁追踪抖音热点并撰写文章，但手动操作太繁琐：

  1. 获取热点数据
  2. 人工筛选安全话题
  3. 撰写3篇文章
  4. 用 DeepSeek 检测文章是否像 AI 写的

这套流程完全可以自动化，于是设计了这个 Skill。

核心设计

设计目标

  1. 一键获取热点：从 API 获取数据，自动写入文档
  2. 安全筛选：过滤政治、黄赌毒、谣言等内容
  3. 真人风格写作：文章看起来像人写的，能通过 AI 检测
  4. 自动审核：调用 DeepSeek 检测，不通过则修改到通过为止
  5. 登录态复用：扫码登录一次，后续自动复用

关键决策

为什么不直接用 MCP 而用 CLI？

方案 优点 缺点
MCP 状态持久化 token 开销大，配置复杂
CLI + Skill 轻量、Claude 原生支持 需要自己处理登录态

选择 CLI + Skill，因为轻量且对 Claude Code 原生友好。

登录态如何处理？

关键点：--headed 参数让浏览器窗口可见，用户可以扫码登录。

详细拆解

工作流程（6步）

第一步：请求热点 API

返回 JSON，解析 data[].word、hot_value、label、video_count 等字段。

第二步：创建当日文件夹

第三步：安全筛选规则

第四步：写作风格

真人风格的关键：

  • 口语化：使用”说实话””其实””感觉”
  • 第一人称：代入个人感受
  • 保留瑕疵：允许轻微重复、思维跳跃
  • 避免 AI 特征：不使用”首先、其次、最后”

第五步：DeepSeek AI 审核

审核逻辑：

  • DeepSeek 判断”人写的”→ 通过
  • DeepSeek 判断”AI写的”→ 按建议修改，重新审核

第六步：登录态管理

关键细节 / 配置

目录结构

热点 API 字段说明

字段 含义
position 排名（1=最热）
word 热点词/话题
hot_value 热度值（整数）
label 标签类型（见下表）
video_count 相关视频数

label 标签类型：

label 含义 安全等级
0 普通话题 需人工判断
1 官方/正能量 最安全
3 社会民生 需人工判断
5 影视娱乐 较安全
8 创意/流行 较安全
9 挑战/互动 较安全
16 辟谣/假新闻 不推荐

设计亮点

  1. 登录态自动复用：用户只需扫码一次，后续全自动
  2. 安全筛选前置：确保产出的内容安全可控
  3. 真人风格写作：专门针对 AI 检测优化
  4. Skill 内嵌博客：读者可以直接提取 Skill 安装

遗留问题 / 可改进方向

  1. 目前仅支持抖音热点，可扩展支持微博、百度等
  2. AI 审核环节可以并行发送以提高速度
  3. 可以增加文章草稿预览环节再审核

如果你想让 AI 永久掌握本文能力

将以下内容保存为 Skill 文件，放入你的 Claude skills 目录，AI 就能在未来的对话中直接调用这项能力，无需每次重新喂文章。

—skill-start—

name: 热点资讯文章撰写
description: |
热点资讯获取 skill。当用户提到以下情况时触发：

  • 获取抖音/微博/百度等平台热点榜单
  • 查看今日热点、热点资讯
  • 热点速递、热点追踪
  • 热点榜单整理并保存到文件
  • 从 API（如 v2.xxapi.cn）获取热点数据

本 skill 完成数据获取、文档写入、安全筛选、文章撰写全流程。
triggers:
- 获取抖音热点
- 查看热点榜单
- 热点资讯
- 热点速递
- 热点追踪

热点资讯获取与安全筛选

本 skill 从抖音热点 API 获取实时数据，写入以日期命名的文件夹，筛选出 3 个安全热点，并为每个热点撰写一篇通过 AI 检测的真人风格文章。

工作流程

第一步：请求热点 API

第二步：创建当日文件夹并写入榜单文档

文件夹路径：当前工作目录/YYYY-MM-DD/
榜单文档路径：当前工作目录/YYYY-MM-DD/热点榜单.md

第三步：安全筛选

从热点列表中筛选出 3 个安全热点：

  • 排除：政治敏感、违法、黄暴恐、谣言（label=16）、企业负面
  • 优先：label=1（官方正能量）、label=5（影视娱乐）、label=8/9（创意流行）

第四步：输出热点榜单（对话中展示）

第五步：为3个安全热点各写一篇文章

文件命名：热点A_[话题关键词].md、热点B_*.md、热点C_*.md

写作风格要求：

  • 口语化：使用”说实话””其实””感觉”
  • 第一人称：代入个人感受
  • 保留瑕疵：允许轻微重复
  • 避免 AI 特征：不使用”首先、其次、最后”

第六步：DeepSeek AI检测审核

审核流程：

  1. 检查登录态文件：当前工作目录/.deepseek-auth.json
  2. 打开浏览器：
    • 有登录态：使用 --profile 加载持久化 profile 并加载登录态
    • 无登录态：使用 --profile 和 --headed 参数打开浏览器，让用户可见并扫码登录
  3. 处理登录（如果浏览器打开后页面在 sign_in 页面）：
    • 提示用户用微信扫码登录
    • 用户登录成功后，保存登录态：playwright-cli state-save 当前工作目录/.deepseek-auth.json
    • 然后重新加载登录态：playwright-cli state-load 当前工作目录/.deepseek-auth.json
  4. 验证登录：确认页面已跳转到主聊天页面
  5. 对每篇文章发送审核请求：
    • 请求内容："请帮我判断下面这篇文章是AI写的还是人写的，并给出修改建议让它更像人写的：[文章内容]"
  6. 根据 DeepSeek 回复判断：
    • **如果回复”AI写的”**：根据修改建议修改文章，然后重新发送审核
    • **如果回复”人写的”**：直接通过
  7. 三篇文章全部审核通过后，保存登录态：playwright-cli state-save 当前工作目录/.deepseek-auth.json
  8. 关闭浏览器

–headed 参数说明：

  • --headed 让浏览器窗口可见，用户可以扫码登录
  • --profile 指定持久化 profile 目录
  • 配合使用：playwright-cli open --profile="当前工作目录/.deepseek-profile" --headed https://chat.deepseek.com/

关键字段说明

字段 含义
code 请求状态码，200 = 成功
data[].position 热点排名
data[].word 热点词/话题
data[].hot_value 热度值（整数）
data[].label 标签类型
data[].video_count 相关视频数量

label 标签类型：

label 含义 安全等级
0 普通话题 需人工判断
1 官方/正能量 最安全
3 社会民生 需人工判断
5 影视娱乐 较安全
8 创意/流行 较安全
9 挑战/互动 较安全
13 人物/明星 需人工判断
16 辟谣/假新闻 不推荐

注意事项

  • 如果 API 请求失败，返回错误信息并提示用户
  • 热度值转换为”万”单位（如 12107641 → 1210万）
  • 文件夹命名格式：YYYY-MM-DD
  • 热点榜单文件名：热点榜单.md
  • 文章文件名：热点A_[话题关键词].md
  • 安全筛选是必做步骤，不可跳过
    —skill-end—
